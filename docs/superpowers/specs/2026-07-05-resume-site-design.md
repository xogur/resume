## Problem Statement

Build a concise Next.js resume site for Kwon Taehyuck, a career backend developer. The site will be linked from hiring platforms such as JobKorea and Saramin, so it must communicate credibility, backend relevance, and production impact within seconds.

The primary readers are:

- Recruiters scanning for role fit, contact information, and career signal.
- Backend engineering interviewers scanning for production experience, technical judgment, and interview topics.

## What Makes This Cool

This is not a decorative portfolio. It is an impact-first web resume.

The first impression should be:

> Java/Spring based practical backend developer with production experience in DB migration, batch automation, monitoring, CI/CD/NCP infrastructure, concurrency control, and realtime AI conversation APIs.

The page should make a recruiter think "this person fits backend server roles" within 10 seconds and make an engineering interviewer want to ask about specific decisions within 30 seconds.

## Constraints

- First version is resume-only. Portfolio pages, blog integration, and long project storytelling are deferred.
- Visual style is clean hiring resume style: white background, restrained accent color, strong typography, compact sections.
- The first viewport must show identity, positioning, contact paths, and quantified impact.
- Public contact information is allowed, including phone number and birth date.
- Content source is the Canva resume draft `권태혁_백엔드개발자_이력서_재구성본`.
- Build as a Next.js project in the current folder.
- Use structured resume data rather than scattering content through components.
- Mobile rendering is required because hiring links may open from mobile job apps.

## Premises

- Hiring-platform visitors scan before they read.
- Operational experience is more persuasive than generic technology lists.
- The resume must show both reliability work and modern realtime AI API work.
- Numbers create attention, but each number needs a short method note to show technical judgment.
- The site should be easy to extend later into portfolio detail pages without rebuilding the content model.

## Approaches Considered

### Approach A: Impact Resume

Lead with name, current role, positioning, contact links, and four quantified proof points. Then present concise sections for core strengths, experience, projects, skills, and contact.

This approach keeps the resume fast to scan while using web layout to make the strongest signals impossible to miss.

### Approach B: Classic Resume

Translate a PDF resume into a web page with traditional sections and minimal visual hierarchy.

This is safe and familiar, but it underuses the web page and may not create enough impact in the first 10 seconds.

### Approach C: Mini Portfolio Resume

Make project stories the center of the page, with richer narrative detail and visual cards.

This may impress engineering interviewers, but it exceeds the current scope. Portfolio expansion is intentionally deferred.

## Visual Design Requirements

- Use a white or near-white page background.
- Use a restrained professional accent color such as blue, teal, or ink.
- Avoid dark terminal styling for v1.
- Avoid decorative hero imagery, large marketing sections, and oversized cards.
- Keep cards or panels compact with border radius of 8px or less.
- Make text readable on mobile without overlapping or horizontal scrolling.
- Use clear visual hierarchy: hero first, metrics second, detailed resume sections after.
- Use icons only where they improve scanning, such as contact links or section labels.

## Technical Design

Use a Next.js App Router application.

Recommended structure:

- `src/app/page.tsx`
  - Renders the single resume page.
- `src/app/layout.tsx`
  - Defines metadata and root layout.
- `src/app/globals.css`
  - Defines global styling and responsive layout.
- `src/data/resume.ts`
  - Exports typed structured resume content.
- `src/components/resume/Hero.tsx`
  - Renders identity, contact links, and proof points.
- `src/components/resume/Section.tsx`
  - Shared section wrapper.
- `src/components/resume/Experience.tsx`
  - Renders career experience.
- `src/components/resume/Projects.tsx`
  - Renders project entries with Problem / Action / Result.
- `src/components/resume/Skills.tsx`
  - Renders grouped skill lists.
- `src/components/resume/Contact.tsx`
  - Renders public contact information.

The app should remain static for v1. No backend API, database, authentication, CMS, or runtime personalization is needed.
