# Resume Site Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a clean impact-first Next.js resume site for Kwon Taehyuck that communicates Java/Spring backend fit and production impact within seconds.

**Architecture:** Use a static Next.js App Router single-page application. Keep resume content centralized in typed structured data and render it through focused resume components. Use CSS modules/global CSS only; avoid a UI framework for v1.

**Tech Stack:** Next.js App Router, TypeScript, React, CSS, npm.

## Global Constraints

- First version is resume-only. Portfolio pages, blog integration, and long project storytelling are deferred.
- Visual style is clean hiring resume style: white background, restrained accent color, strong typography, compact sections.
- The first viewport must show identity, positioning, contact paths, and quantified impact.
- Public contact information is allowed, including phone number and birth date.
- Content source is the Canva resume draft `권태혁_백엔드개발자_이력서_재구성본`.
- Build as a Next.js project in the current folder.
- Use structured resume data rather than scattering content through components.
- Mobile rendering is required because hiring links may open from mobile job apps.
- The app remains static for v1. No backend API, database, authentication, CMS, runtime personalization, analytics, or contact form.
- This workspace is not currently a git repository. Commit steps are included for repositories but should be skipped unless git is initialized before execution.
