## Problem Statement
Build a concise Next.js resume site for Kwon Taehyuck, a career backend developer. The site will be linked from hiring platforms such as JobKorea and Saramin, so it must communicate credibility and fit within seconds.

The primary reader is a recruiter or backend engineering interviewer deciding whether this Java/Spring backend developer is worth contacting.

## What Makes This Cool
This should not feel like a decorative portfolio. It should feel like a sharp web resume: fast to scan, concrete, and backed by production numbers.

The strongest impression is:

"Java/Spring based practical backend developer with production experience in DB migration, batch automation, monitoring, CI/CD/NCP infrastructure, concurrency control, and realtime AI conversation APIs."

## Constraints
- First version is resume-only. Portfolio expansion comes later.
- Visual style: clean hiring resume style, white background, strong typography, clear sections.
- Public contact information is allowed, including phone number and birth date.
- Content source is the Canva resume draft `권태혁_백엔드개발자_이력서_재구성본`.
- The first screen must create impact quickly; avoid long storytelling.
- Build as a Next.js project in the current folder.

## Premises
- Hiring-platform visitors scan before they read.
- The first screen should prove backend relevance with role, focus areas, and measurable outcomes.
- Operational experience is more persuasive than generic technology lists.
- AI/realtime API experience should be visible early because it differentiates the resume from a standard Spring backend profile.

## Approaches Considered
### Approach A: Clean Resume Site
White background, compact sections, recruiter-friendly structure, and measurable highlights near the top. This is the best fit for JobKorea/Saramin links.

### Approach B: Developer Profile Site
Darker, more technical, with terminal-like cues and stronger personality. This may impress engineers but is less immediately readable for recruiters.

### Approach C: Portfolio Showcase
Project stories become the main content. This is useful later, but it expands scope beyond the first resume version.

## Recommended Approach
Use Approach A: Clean Resume Site.

The page should prioritize scan speed and trust. It should lead with name, role, concise positioning, contact links, and four quantified proof points:

- 10M+ row production DB migration
- Up to 97% query response time reduction
- 17 Airflow DAGs for operational automation
- Under 500ms average TTS playback latency after user speech in an STT -> LLM -> TTS realtime conversation pipeline
